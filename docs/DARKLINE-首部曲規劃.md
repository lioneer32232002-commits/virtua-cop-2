## 1. MVP 定義（最小可行＝遊戲的「起點」）

**一句話：** 一關可從頭玩到尾的 1950s 台北諜報軌道射擊，含開場簡報 → 軌道戰鬥（3 arena + boss）→ 結尾留 1996 伏筆，外加一個極簡「情報層」過場。

**MVP 必須有（Must）：**
- 1 個原創場景 preset：**1950s 台北騎樓街/中山北路**（程序生成，先不靠 Gemini 也能跑）
- 1 條完整關卡：軌道運鏡 + 3 個 arena 波次 + 1 個 boss + 至少 1 名「平民/人質」（誤擊扣分，沿用現有 innocent 機制）
- 玩家武器：**M1911 手槍**（沿用現有 `WeaponViewModel`，外觀微調為 1911 剪影）
- 敵人：持「burp gun」剪影的特務/便衣（沿用程序人形 + 服裝配色區分陣營）
- 開場/結尾**簡報畫面**（純文字 + 一張背景圖即可）建立故事與伏筆
- 1 段極簡**情報層互動**（任務間：一個解碼/拼線索的小互動，證明「諜報層」概念）
- 選單能選到這關（`LevelLoader` 自動發現，現成）

**MVP 先不做（Won't，列入之後）：** VR、手機觸控、多關、複雜情報系統、原創角色動補、配音、BGM 編曲（先用合成音/版權音樂佔位）。

**驗收標準：** 從選單選關 → 完整玩通一輪（簡報→戰鬥→伏筆）→ 流暢無卡頓（沿用現有效能管線）→ Web 瀏覽器與 Electron 包裝皆可跑。

## 2. 首部曲故事框架（虛構化）

> **史實靈感**（見市場報告 §5）：1950–55 駐台美軍 MAAG、第七艦隊協防、CIA 前線「西方公司（Western Enterprises）」以民間商號為掩護、自台灣與外島支援對大陸沿海游擊；反共義士（1.4 萬戰俘赴台、刺青、一二三自由日、駕機投誠）。**全部虛構化使用。**

**設定（虛構）：**
- 年代：**1953 年，台北**（韓戰停戰前夕，駐台美軍與情報活動最濃的時點）。
- 主角（虛構）：**林沂 / 代號「暗線」**——表面是「**西緣貿易公司**」（虛構，影射西方公司）的物流顧問，實為替美方與國府牽線的本土情報員。
- 反派陣營（虛構）：**「北方某情報機構」滲透台北的地下網**（不具名、不影射特定真實機關），目標是策反/暗殺一名即將揭露情報網的關鍵人物。
- 麥高芬：一名身負「義士刺青」的投誠者（虛構），其實是雙面間諜，握有一份能掀翻台北情報網的名單（「暗線名單」）。

**首部曲一關的劇情拍子（對應軌道流程）：**
1. **開場簡報**：西緣貿易公司閣樓，林沂受命在騎樓街區攔截一場交接——「北方的人」要在今晚帶走那份名單。
2. **Arena 1（騎樓街）**：便衣特務伏擊，軌道推進清場。
3. **Arena 2（市場/巷口）**：人質情境——那名投誠者出現（平民模型，誤擊重罰），玩家須在交火中保住他。
4. **Arena 3（碼頭倉庫/中山北路美軍區外圍）**：追上交接點。
5. **Boss**：北方網的行動指揮（虛構），擊退後奪回名單。
6. **結尾伏筆（鉤到 1996）**：名單上有一個被劃掉的代號與一行字——「**待第一島鏈再次收緊之日**」。鏡頭/字卡跳到 **1996 年台海**：飛彈試射、鎖台軍演、首次總統直選在即（市場報告 §8）。**暗示主角的諜報血脈將由後人在 1996 接續「保護民選元首」。**

**為何這個故事好：** 真實史骨架（駐台美軍 + CIA 前線 + 投誠者）戲劇張力足、視覺可辨識（騎樓、吉普、刺青、burp gun vs 1911），又因官方細節稀少而**留給虛構極大自由**（市場報告 §5 已標註西方公司細節未公開＝對虛構是好事）。「暗線名單 + 第一島鏈再次收緊」一句就把首部曲與 1996 第二部曲縫起來。

## 3. 場景設計：1950s 台北騎樓街（新 preset）

**做法＝複製 downtown1 套路，新增一個 `OriginalEnvironment` preset「`taipei1950s`」。**（現有 `OriginalEnvironment.js` 已是 seeded 程序生成、`StageEnvironment` 在無 SEGA chunk 時走原創環境——架構現成，只加 preset。）

**程序生成要素（對應老照片考據，市場報告 §6）：**
- **騎樓街**：雙側連續柱列 + 頂蓋走廊（取代 downtown 的純大樓盒）；招牌掛在騎樓樑下。
- **街面**：柏油/水泥路 + 側溝（細長 mesh）；`groundYAt` 落地面 y=0（沿用現有落地邏輯）。
- **物件**：三輪車、停放的吉普、腳踏車、冰淇淋攤（低多邊形 primitive 即可）。
- **招牌**：直式中文字招牌（先用程序貼圖/色塊，之後 Gemini 升圖）。
- **天空**：黃昏/陰天暖色 dome（沿用現有 `sky.js` 漸層 + fog）。
- **氛圍道具**：晾衣繩、電線桿、美軍區鐵柵門（Arena 3）。

**運鏡：** 自編 `railPath` curve（沿用 downtown1 的 `rigMode:curve` + `railDuration`），讓相機沿騎樓街推進，在 3 個開闊點停成 arena（`clearPoints`）。**dwell 點務必挑開闊無遮蔽處**（騎樓柱會擋 spawn——記取 stage2 貼牆遮 spawn 的坑）。

## 4. 美術與資產策略（精緻但輕巧 + 流暢）

**兩段式（先能跑、再升質）：**
1. **第一段（程序生成，現成）**：primitive + 程序貼圖 + 亮窗，先把整關跑通、波次調好。容量幾乎為零、流暢。
2. **第二段（Gemini 升圖）**：對**貼圖**而非幾何升質——用 Gemini 生「1950s 台北招牌/磚牆/木門/黃昏天空/海報」貼到程序幾何上。質感大升、容量仍輕（幾張 texture）。**生圖時避開任何「VC 風格」字樣與 SEGA 角色即乾淨**（已知守則）。

**流暢感保障（已驗證的引擎資產）：**
- draw-call 合併（`render/mergeStatic.js`）已實測 **~9× 快**（11442→1461 calls）——新 preset 同樣受惠。
- 美術走「**輕量幾何 + 升圖貼圖**」路線：用貼圖表現細節（招牌、磚紋、鏽蝕），而非堆三角面。容量輕、手機/Web 也跑得動。
- 軌道射擊本質上相機受控 → 可只對「相機會看到的扇形」下美術功夫（之後可做視錐剔除/離線預 merge GLB，roadmap 已列）。

## 5. 武器與服裝

**武器（市場報告 §7）：**
- **玩家：M1911 手槍**——沿用現有 `WeaponViewModel`（目前是程序化 box+cylinder 手槍、後座力真實），把剪影微調為 1911（滑套/握把比例）即可，**不必先做精細模型**。
- **敵人：PPSh-41 / Type 50「burp gun」剪影**——彈鼓圓盤是極強的時代/陣營辨識符號，低多邊形也認得出。
- 取捨：軌道射擊不需武器庫，**1 主武器 + 敵人剪影對比**就夠建立 1953 冷戰質感。

**服裝（沿用程序人形 + 配色/部件區分，市場報告 §6）：**
- 主角/友方：西裝（情報員）、美軍卡其制服、國軍軍裝。
- 敵方便衣：深色西裝/中山裝剪影。
- 平民/人質：**旗袍**（女性）、布衫/工人服（男性）、投誠者（軍便服 + 可見刺青貼圖＝伏筆視覺鉤）。
- 現有 `CharacterFactory` 程序人形（含命中區）足以承載——先用配色/簡單部件區分陣營，原創角色美術列為之後。

## 6. 情報/解謎層（諜報層的 MVP 證明）

市場報告 §1 指出「諜報敘事」級距比純光槍高一截——**這層是 DARKLINE 區別於普通光槍作的關鍵賣點**，MVP 要放一個「最小但能成立」的版本：

- **任務間一個解碼小互動**：例如截獲的電碼/暗號，玩家把碎片線索拼出「交接地點」才解鎖下一段軌道。
- 技術上＝一個 UI/互動場景（HTML overlay 或 three.js 場景皆可，引擎已能做 overlay/簡報）。
- **MVP 只要一個**，證明「動作層 + 情報層」雙層結構成立即可；完整情報系統列入之後。

## 7. 技術落地步驟（對齊現有檔案，可直接接手）

> 以下是「複製 downtown1 套路」的具體清單，每步都對應現有架構：

1. **新場景 preset**：在 `game/src/scene/OriginalEnvironment.js` 加 `taipei1950s` preset（騎樓柱列、側溝、三輪車/吉普 primitive、直式招牌位）。沿用 seeded PRNG（可單元測試）。
2. **新關卡 JSON**：複製 `game/src/level/levels/downtown1.json` → `taipei1950s.json`，改 `id`、`environment{type:"taipei1950s",seed}`、自編 `railPath`/`railDuration`、3 arena + boss + 人質波次、對齊 `clearPoints`。**自動上選單**（`LevelLoader` glob 現成）。
3. **dwell 點目視校正**：`analyze-path` + preview 截圖，挑騎樓間的開闊點當 arena（避開柱子遮 spawn）。
4. **玩家槍剪影**：微調 `game/src/weapon/WeaponViewModel.js` 比例為 1911（小 cut）。
5. **敵人陣營配色 + burp gun 剪影**：在 `CharacterFactory`/`EnemyManager` 層用配色/簡單部件區分；burp gun 用低多邊形彈鼓剪影。
6. **開場/結尾簡報 + 情報層 overlay**：HTML overlay 或簡報場景（引擎已有 overlay 機制）。結尾字卡跳 1996 伏筆。
7. **（之後）Gemini 升圖**：生招牌/牆面/天空貼圖貼上程序幾何。
8. **（上架前）Electron 包裝 + steamworks.js**：市場報告 §2 已驗證路徑（MIT、維護中）。
9. **測試**：沿用 `cd game && npx vitest run`（preset 的 seeded 生成、關卡載入、波次都可寫測試，照現有 +N 測試慣例）。

**所有改動只動原創新檔 + 必要 hunk，不碰 stage1/2/3 與 CharacterFactory 敵人主線**（沿用既有隔離原則）。

## 9. 題材風險控管（務必遵守）

承市場報告 §3（《還願》教訓）：
- **全面虛構化**：人名（林沂、暗線）、組織（西緣貿易公司、北方某情報機構）、地點（虛構街名）一律不具名、不影射特定真實機關/人物。
- **免責聲明**：片頭加「本作純屬虛構，如有雷同純屬巧合」。
- **商業模型從第一天排除中國市場**：不送 Steam China、不為過審改題材；主打台灣 + 海外華語 + 全球。
- **不埋政治彩蛋**：《還願》正是栽在隱藏符咒——DARKLINE 不放任何可被解讀為影射在世政治人物的隱藏內容。
- **把地緣張力當「戲」不當「政治宣言」**：賣的是冷戰諜報的戲劇感，不是真實政治立場。
