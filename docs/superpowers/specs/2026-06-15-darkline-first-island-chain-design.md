# 《暗線 / DARKLINE — First Island Chain》設計規格（spec）

> 產出日期：2026-06-15
> 狀態：設計定案，待寫實作計畫（writing-plans）
> 上游文件：[市場研究報告](../../DARKLINE-市場研究報告.md)、[首部曲規劃](../../DARKLINE-首部曲規劃.md)
> **本 spec 取代「首部曲規劃」中的「純軌道換皮 downtown1」玩法假設**——經 2026-06-15 腦力激盪，玩法演進為「半軌道＋半自由的 sprite 射擊冒險」（詳見 §2）。市場結論、題材、風險控管仍承襲上游文件。

## 1. 概念一句話

**1953 台北的兩岸諜報，採 sprite（2D 貼圖）美術的「半軌道＋半自由」射擊冒險。**

精神對標：玩法氣質取自 **Snatcher / Policenauts**（諜報調查＋射擊段落）＋ **Duke3D / Ion Fury**（sprite 美術、跑得動在爛機器上的 boomer-shooter 質感）。題材取自韓戰駐台史骨架（全面虛構化）。

定位（承市場報告）：Steam（PC，Electron 包裝）優先的短而精精品；US$9.99–14.99；全面虛構化、放棄中國市場；題材獨特性當行銷資產；Web 免費試玩導流。

---

## 2. 玩法結構：半軌道＋半自由（核心決策）

一個「任務」由三種段落交織：

| 段落 | 內容 | 引擎負擔 |
|---|---|---|
| **🎯 軌道段** | 相機沿 curve 自動推進的街機射擊（伏擊、追逐、Boss） | 沿用現有引擎，零敵人 AI |
| **🚶 自由段** | 小範圍可走動空間：調查、找情報、輕解謎、與會走動的敵人交火 | 新增「受限第一人稱」相機 + 簡單 AI + 碰撞 |
| **📋 簡報段** | 任務間劇情 + 情報解碼小互動 | UI/overlay |

### 2.1 部曲式風險階梯（關鍵策略）

- **首部曲**：軌道:自由 ≈ **50:50**（實際比例由 M0 結果決定，見 §4）。
- **比例是可調旋鈕**：首部曲完成、確認自由段的 AI／碰撞／多角度 sprite 技術可克服後，**第二部曲把自由比例往上調**，逐步逼近「全自由 Duke3D」。
- 原則：**每一部曲都是下一部的墊腳石，永遠不押上全部身家。**

### 2.2 為何不一次做全自由（已評估並否決為「首發即全開放」）

全自由 FPS 會多出：敵人 AI（地圖中追擊/繞牆/掩護）、完整可走地圖設計、全方位碰撞、玩家物理、**每個敵人 5–8 角度 sprite**（讓 §3 的頭號風險惡化數倍），且會讓現有軌道引擎大半失效。獨立/個人 FPS 最常死在「敵人 AI」與「關卡設計」這兩個無底洞。故採「半自由起步、部曲遞增」而非「首發全開放」。

## 3. 美術路線：sprite + 程式化風格收斂

- **角色（敵人、平民）= 2D sprite**（公告板）：Gemini 生圖直接用，避開 3D 建模/綁骨/動補（即舊 H 階段 MOT 骨架地獄）。
- **場景 = 輕量幾何 + 貼圖**：沿用 `OriginalEnvironment` 程序生成 + draw-call 合併（已驗 ~9× 快），低配筆電可跑。
- **不追求逼真**，走 stylized 復古；正中「Duke3D 流暢感 + Gemini 美術」同一條路線。

### 3.1 風格一致性方案（頭號技術風險的對策）

Gemini「同樣指令會吐不同風格」是已知問題。**一致性不靠 Gemini 自律，靠後處理管線強制收斂**（程式、確定性）：

1. **🔑 鎖定共用調色盤**（最強）：所有 Gemini 輸出強制量化到一個固定 32/64 色調色盤 + 統一描邊 + 統一解析度。等同 Duke3D 全美術共用 256 色盤的做法。即使 Gemini 風格在抖，過完同一管線即收斂。
2. **參考圖鎖定（image-to-image）**：先定一張「風格聖經」master 圖，之後全部圖生圖。
3. **固定 style 咒語片段**：色調 hex／硬陰影／線條粗細／「1950s pulp、有限色盤」每次原封套用。
4. **一張圖一次生整組**（多角度/多動畫格畫在同一張，再程式切割）。
5. **人工挑 + 重生離群值**。

> ⚠️ **形狀/姿勢一致性（跨動畫格「同一個人」）是調色盤解不了的殘留風險**，列為 M0 第一個要驗的東西（§4）。緩解：動畫格數壓少（2–3 格、復古味）、圖生圖鎖底稿、必要時人工描修。

## 4. M0 垂直切片（先打地雷，丟棄式）

**做任何正式內容前**先做醜原型，只為證實四個未知數。**M0 是整個案子最重要的一步——把最大賭注排到第一週、用玩具驗。**

| 要驗 | 通過標準 |
|---|---|
| ① Gemini 生**一個**敵人過調色盤管線 → sprite 風格+多角度+動畫一致性 | 能產出一個風格一致、能動的敵人 |
| ② 一個小房間：走動+碰撞+該敵人簡單 AI+射擊 → 自由段好不好玩、AI 多痛 | 自由段交火「好玩且工程可控」 |
| ③ 軌道→自由→軌道→存檔 完整迴圈 | 一輪跑通、存檔可讀回 |
| ④ Electron 打包延遲/體積 | 射擊延遲可接受 |

**決策閘（M0 之後）：**
- 自由段又好玩又做得動 → 首部曲自由比例可拉高，往全自由路線鋪路。
- 自由段是無底洞 → 縮小自由段、軌道為主幹，C 安全網生效。
- **首部曲軌道:自由比例由此決定，不靠事前猜測。**

## 5. 首部曲 MVP 範圍（＝你要的「遊戲起點」）

一條從頭玩到尾的任務（1953 台北）：

1. **開場簡報**：西緣貿易公司閣樓，林沂受命攔截名單交接（劇情+目標）。
2. **🎯 軌道段 1**：騎樓街便衣伏擊。
3. **🚶 自由段**：市場/巷弄——走動找線索、保護投誠者（沿用 innocent 誤擊扣分）、輕度交火。
4. **🎯 軌道段 2 / Boss**：碼頭對決，奪回「暗線名單」。
5. **📋 結尾字卡**：1996 伏筆鉤子（「待第一島鏈再次收緊之日」）。

- 玩家武器：**M1911**（沿用 `WeaponViewModel`，剪影微調）。
- 敵人：**burp-gun（PPSh/Type 50）特務 sprite**。
- **存檔系統**（localStorage/檔案）。
- **中英文**從第一天做（§8）。

**故事框架（虛構化，承首部曲規劃 §2）：** 主角林沂/代號「暗線」（虛構物流顧問實為情報員）、西緣貿易公司（影射西方公司）、北方某情報機構（不具名）、麥高芬＝身負義士刺青的雙面投誠者握「暗線名單」。**全部虛構，片頭加「純屬虛構」免責。**

### 5.1 MVP 先不做（YAGNI）

VR、手機觸控、配音、原創 BGM 編曲（先用免費庫佔位）、多任務、複雜情報系統、全自由大地圖、敵人複雜 AI（M0 只驗最簡 AI）。

## 6. 技術架構

- **共用渲染器/場景**：軌道段與自由段**共用同一個 three.js 渲染器與場景**，只切換「相機由誰控制」——軌道＝curve 自動推進（現成）；自由＝受限第一人稱（PointerLock，刻意做簡單：無跳躍、無複雜物理、平地走動+看+互動+射擊）。
- **隔離原則**：兩模式邊界（下車→走動→上車）保持簡單，狀態切換清楚。
- **滑鼠瞄準**：把 lock-on 爆頭手感移植到滑鼠（靈敏度/輔助力度要調，列入 M1）。
- **效能**：sprite 比 3D 模型輕；目標低配筆電。沿用 draw-call 合併。
- **存檔**：段落進度 + 分數，標準序列化。
- **i18n**：所有文字抽成語言檔，程式不寫死字串。

## 7. 資料夾分法（VRCOP / 原創切開）

repo 現況有三層歷史沉積，目標：**出貨時一眼能判斷「哪些絕對不能進 build」（避免 SEGA IP 污染）。**

| 類別 | 現況位置 | 目標 |
|---|---|---|
| **舊 Unity 專案（已棄用）** | `Assets/`、`ProjectSettings/`、`Packages/` | 歸檔到 `legacy-unity/`（或獨立分支/封存），明確標「已棄用」 |
| **VRCOP 舊資產/工具** | `game/src/level/levels/stage1-3.json`、`tools/extract-stage-assets`、SEGA 萃取物、MOT 實驗 | 收進 `legacy-vrcop/`，標「參考用、含 SEGA 衍生物、**永不出貨**」 |
| **共用引擎** | `game/`（軌道、渲染、效能管線） | 留原地，兩邊共用 |
| **DARKLINE 原創** | `downtown1.json`、未來 `taipei1950s`、原創 sprite/劇情 | 收進 `darkline/` 命名空間 |

> 精確的檔案搬移清單在實作計畫階段定（會掃一次 import 相依，避免搬壞引用）。本 spec 只定原則。

## 8. 免費可商用資源（已確認，含授權紀律）

**字體：** 思源黑體/宋體（Noto CJK TC，OFL）為安全底盤；jf-openhuninn（粉圓）、**Cubic 11（點陣繁中，搭 sprite 復古味）** 為風格選項；英文像素字 Press Start 2P / m5x7 等（OFL）。
**音效：** Kenney.nl、freesound.org（篩 CC0）、Sonniss GDC Bundle。
**音樂：** incompetech（Kevin MacLeod）、OpenGameArt（多為 CC-BY）。

**授權紀律（硬規則）：**
- 只用 **CC0**（無條件）或 **CC-BY**（須致謝）或 **OFL**（字體）。**CC-BY-NC（禁商用）絕對不用。**
- **第一天建 `CREDITS` 清單**，每用一個素材登記授權與出處。無現金成本，有記帳成本。

**中英文做法：** 文字抽語言檔（繁中+英文）；含字招牌用獨立圖層或盡量用無字招牌；MVP 不做配音。

## 9. 開發里程碑（粗估級距，非承諾日期）

⚠️ 依投入頻率而定，且 M0 結果可能重塑後續。

| 里程碑 | 內容 | 粗估 |
|---|---|---|
| **M0** | 垂直切片打地雷（§4） | ~1–2 週 |
| **M1** | 可玩骨架：1 任務軌道+自由跑通（佔位美術）+ 滑鼠手感 | ~3–4 週 |
| **M2** | 劇情+情報+武器/敵人 sprite+存檔+中英文 → **＝ MVP** | ~3–4 週 |
| **M3** | Gemini 美術升質 + 音效 + 補成完整首部曲 | ~1–2 月 |
| **M4** | Electron+Steam 包裝 + Web 試玩 + 商店頁（願望單） | ~2–4 週 |

**M0+M1+M2 = 能玩、能截圖、能放 Web 的「首部曲起點」。**

## 10. Steam 成本效益與上架門檻

- **現金成本：極低。** Steam Direct 上架費 **US$100（一次性、可回收）** + 免費素材庫（$0）。真正成本＝時間。
- **品質門檻（要有人買）：** 射擊手感扎實 + 流暢 + 美術風格一致（調色盤）+ 商店頁一句話勾子（「台灣冷戰諜報・sprite 射擊」）+ 內容 2–4hr 或精煉 demo + **上架前開頁累積願望單**。
- **效益（務實）：** 不會發財。成功＝回本 + 累積觀眾 + 留下可重用技術/美術資產。boomer-shooter 是活市場（Ion Fury 為證），題材獨特性是行銷資產。

## 11. 風險登記簿（誠實版）

| 風險 | 等級 | 對策 |
|---|---|---|
| Gemini 跨動畫格「同一個人」形狀一致性 | 🔴 最高 | M0 第一個驗；壓少格數、圖生圖鎖底稿、人工描修 |
| 自由段 AI/碰撞變無底洞 | 🟡 | M0 驗；做不動就退軌道為主（安全網） |
| Electron 延遲/體積 | 🟡 | M0 打包 smoke test |
| 滑鼠瞄準手感 | 🟡 | M1 調靈敏度/輔助 |
| 素材授權污染 | 🟢 | CC0/CC-BY/OFL 限定 + CREDITS 記帳 |
| 低配效能 | 🟢 | sprite 輕 + draw-call 合併；目標機實測 |

**沒有架構級硬傷；唯一真正的技術未知數是「Gemini sprite 動畫一致性」，已排為 M0 第一驗證項。**

## 12. 驗收標準（首部曲 MVP）
